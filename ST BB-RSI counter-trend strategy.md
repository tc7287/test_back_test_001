# 📈 주식 자동 매매: BB-RSI 역추세 전략 (Mean Reversion)

주식 자동 매매의 세계에 오신 것을 환영합니다! 감이 아닌 **'명확한 수치'**와 **'일관된 규칙'**으로 매매하는 것이 핵심입니다. 이 전략은 변동성을 측정하는 **불린저 밴드(Bollinger Bands)**와 과매수/과매도를 판단하는 **RSI(상대강도지수)**를 결합하여 승률을 높인 역추세 매매 기법입니다.

---

## 1. 전략 개요 (Strategy Concept)
*   **성격**: 평균 회귀 (Mean Reversion) - 고무줄이 당겨졌다가 제자리로 돌아오듯, 과하게 튀어나간 주가를 잡아내는 방식입니다.
*   **핵심 원리**: 주가가 통계적 변동 범위(BB 하단)를 벗어나고, 심리적으로도 과매도(RSI 침체) 상태일 때의 기술적 반등을 노립니다.
*   **적정 주기**: **15분봉 또는 1시간봉** (데이 트레이딩 및 스윙에 최적)
    *   *주의: 현재 시스템은 일봉(Daily) 데이터를 기반으로 백테스트를 수행합니다.*

---

## 2. 설정 파라미터 (Parameters)

| 구분 | 항목 | 설정값 | 비고 |
| :--- | :--- | :--- | :--- |
| **Bollinger Bands** | 기간 (Period) | **20** | 이동평균선 기준 |
| | 표준편차 (StdDev) | **2** | 가격의 약 95.4%가 이 밴드 안에 위치 |
| **RSI** | 기간 (Period) | **14** | 기본 설정값 |
| | 과매수 기준 | **70** | 하락 반전 가능성 (70 이상) |
| | 과매도 기준 | **30** | 상승 반전 가능성 (30 이하) |

---

## 3. 매수 및 매도 타이밍 (Trading Logic)

### ✅ 매수 타이밍 (Buy Signal)
다음 두 조건이 **동시에 만족(AND)**될 때 매수합니다.

1.  **가격 조건**: 현재가(Close) ≤ 불린저 밴드 **하단선 (Lower Band)**
2.  **보조 조건**: RSI(14) ≤ **30** (과매도 구간)

> **이유**: 단순히 밴드 하단에 닿았다고 사는 것이 아니라, '심리적으로도 팔 만큼 팔았다'는 신호가 올 때까지 기다려 승률을 높입니다.

### ✅ 매매 청산 (Exit Strategy)
익절과 손절은 철저하게 지켜야 합니다.

1.  **익절 (Take Profit)**
    *   **1차 (보수적)**: 가격이 불린저 밴드 **중심선 (20일 이동평균선)**을 상향 돌파 시
    *   **2차 (공격적)**: 가격이 불린저 밴드 **상단선 (Upper Band)**에 도달 시
2.  **손절 (Stop Loss)**
    *   진입가 대비 **-2% ~ -3%** 이탈 시 즉시 매도 (필수!)
    *   또는 전저점 이탈 시

---

## 4. 수식 요약 (Technical Background)

알고리즘 구현을 위한 기본 수식입니다.

*   **중심선 (MB)**: $SMA_{20}$ (20일 단순 이동평균)
*   **상단선 (UP)**: $MB + (2 \times \sigma)$
*   **하단선 (DN)**: $MB - (2 \times \sigma)$
    *   ($\sigma$: 종가의 표준편차)

```python
# Python 구현 예시
upper_band = sma20 + (std_dev * 2)
lower_band = sma20 - (std_dev * 2)

if (current_price <= lower_band) and (current_rsi <= 30):
    execute_buy_order()
```

## 💡 자동 매매 구현 시 주의사항 (Tips)

1.  **횡보장 vs 추세장**: 이 기법은 **박스권(횡보장)**에서 수익률이 극대화됩니다. 강력한 하락 추세가 터지면 밴드를 타고 계속 흘러내릴 수 있으므로, **거래량**이 평소보다 지나치게 터질 때는 매매를 정지하는 필터를 고려하세요.
2.  **데이터 주기**: 본 전략은 15분/1시간봉에 최적화되어 있으나, 현재 백테스트 시스템은 **일봉(Daily)** 데이터를 사용하고 있어 결과가 다를 수 있습니다.
3.  **백테스팅 필수**: 과거 1~2년치 데이터로 반드시 시뮬레이션을 돌려보고, 종목별 변동성에 맞춰 파라미터를 미세 조정하세요.
